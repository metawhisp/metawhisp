# PROP-0001: AudioSource Protocol {#root}

## Назначение {#purpose}

Унифицировать интерфейс источников аудио чтобы `TranscriptionCoordinator` и meeting recording работали с любым источником без branching.

## Инварианты (Critical) {#invariants}

- **Выходной формат ВСЕГДА 16kHz mono Float32 PCM** — это контракт, на который полагается WhisperKit
- **`stop() -> [Float]`** возвращает ВЕСЬ буфер с момента `start()`
- **`isRecording`** становится `true` ТОЛЬКО после успешного старта (async setup должен завершиться)
- **`hasPermission`** — чистый query, БЕЗ побочных эффектов (не триггерит TCC)
- **`requestPermission()`** — может триггерить TCC диалог; возвращает финальный статус

## 1. Протокол {#protocol}

```swift
@MainActor
protocol AudioSource: AnyObject {
    var isRecording: Bool { get }
    var audioLevel: Float { get }
    var audioBars: [Float] { get }
    var hasPermission: Bool { get }
    func requestPermission() async -> Bool
    func start() throws
    func stop() -> [Float]
}
```

**Файл:** `Services/Audio/AudioSource.swift`

## 2. Реализации {#implementations}

### 2.1. Mic {#implementations.mic}

`AudioRecordingService` (`Services/Audio/AudioRecordingService.swift`).

- Источник: AVAudioEngine на default input device
- Формат входа: что отдаёт железо (обычно 44.1/48 kHz)
- Ресемплер: AVAudioConverter → 16kHz mono Float32
- Permission: `AVAudioApplication.shared.recordPermission` (macOS 14+)

### 2.2. System Audio {#implementations.system}

`SystemAudioCaptureService` (`Services/Audio/SystemAudioCaptureService.swift`).

- Источник: `SCStream` с `capturesAudio = true`, `excludesCurrentProcessAudio = true`
- Формат входа: 48kHz stereo Float32 (конфигурируется)
- Ресемплер: mono-mix + linear interpolation → 16kHz mono Float32
- Permission: ScreenCaptureKit TCC (триггерится первым вызовом `SCShareableContent`)

**Почему excludesCurrentProcessAudio:** иначе звуки MetaWhisp (system sound "Start recording" и т.д.) попадают в запись и создают feedback.

### 2.3. Meeting (mix) {#implementations.meeting}

`MeetingRecorder` (`Services/Audio/MeetingRecorder.swift`) — НЕ conform напрямую к AudioSource (использует другой storage), но имеет совместимый API. Оборачивает mic + system.

**Почему отдельный класс:** `TranscriptionCoordinator` не рассчитан на 2 источника с раздельным stop(). Meeting pipeline живёт в `AppDelegate`, не в coordinator.

## 3. Запрет на состояние {#no-state-leak}

Между `stop()` и следующим `start()` реализация НЕ должна держать:
- Ссылки на предыдущий buffer (GC его уже отдал пользователю)
- Активные subscribers на CoreBluetooth / AVAudioEngine / SCStream
- Open file descriptors

Проверяется тем, что повторный `start()` корректно работает без `init()`.

## 4. Acceptance {#acceptance}

- `TranscriptionCoordinator` хранит `any AudioSource` — ✅ реализовано
- Добавление нового источника (например BLE wearable) не требует изменения coordinator — design ✅, не протестировано
- Возврат из `stop()` — массив готов к подаче в WhisperKit без доп. обработки — ✅
