# FEAT-0001: Meeting Recording {#root}

## User story {#story}

**Как** пользователь в созвоне Zoom/Meet/Teams,
**Я хочу** нажать одну кнопку в menu bar → MetaWhisp запишет обе стороны разговора (мой голос + других участников) → после остановки получу полный транскрипт локально.
**Без** отправки аудио в Deepgram/Firebase/OpenAI.

## Acceptance criteria (Critical) {#acceptance}

- Запись стартует по кнопке "RECORD" в попапе menu bar
- Авто-детект созвона: если открыт Zoom/Meet/Teams/FaceTime/Slack — показать "{APP} DETECTED"
- Во время записи: красный индикатор + идущий таймер в попапе
- На stop: транскрипт появляется в Insights → Meetings в течение разумного времени (зависит от длины)
- Галлюцинации Whisper (≈"Продолжение следует") НЕ попадают в сохранённый транскрипт
- Если mic permission нет — запись system audio продолжается, пользователь видит предупреждение
- Никакие данные не покидают Mac (кроме если пользователь выбрал Cloud transcription provider в Settings)

## 1. Архитектура {#architecture}

### 1.1. Pipeline {#architecture.pipeline}

```
User clicks RECORD
    ↓
AppDelegate.toggleMeetingRecording()
    ↓
MeetingRecorder.start()
    ├─ SystemAudioCaptureService.start() (async setup SCStream)
    │    ↓ [waits up to 5s for isRecording=true]
    └─ AudioRecordingService.start() (mic, synchronous)
    ↓
[recording — user speaks, others speak through speakers]
    ↓
User clicks STOP
    ↓
MeetingRecorder.stop() → mixed [Float] samples (§2.3)
    ↓
AppDelegate.stopMeetingRecording()
    ├─ Segment into ≤5min chunks
    ├─ For each chunk:
    │   ├─ RMS < 0.0005 → skip
    │   ├─ coordinator.activeEngine.transcribe(chunk)
    │   ├─ isAlwaysHallucination → skip
    │   └─ isHallucination + RMS < 0.003 → skip
    └─ Save HistoryItem(source: "meeting")
```

### 1.2. Ключевые файлы {#architecture.files}

- `Services/Audio/MeetingRecorder.swift` — оркестратор mic + system
- `App/AppDelegate.swift` § `startMeetingRecording`, `stopMeetingRecording`
- `Views/MenuBar/MenuBarView.swift` § `meetingStrip` — UI в попапе
- `Views/Windows/InsightsView.swift` § `meetingHistoryView` — список транскриптов

## 2. Audio mixing {#meeting-mix}

### 2.1. Проблема {#meeting-mix.problem}

System audio (SCStream) захватывает ТОЛЬКО то что играет из колонок. Голос пользователя в mic → по сети в Zoom → НЕ попадает в speakers → теряется.

### 2.2. Решение {#meeting-mix.solution}

`MeetingRecorder` параллельно держит:
- `AudioRecordingService` для mic
- `SystemAudioCaptureService` для system audio

На `stop()` оба возвращают `[Float]` 16kHz mono. Складываются sample-by-sample:

```swift
for i in 0..<common {
    let sum = mic[i] + system[i]
    mixed[i] = max(-1.0, min(1.0, sum))  // soft clip
}
```

Tail более длинного стрима копируется как есть. Выравнивание по sample 0 — старт обоих сервисов происходит в пределах нескольких сотен ms, для транскрипции не критично.

### 2.3. Graceful degradation {#meeting-mix.degradation}

Если `mic.hasPermission == false` или `mic.start()` бросает:
- `micOnlyMode = true`
- Запись продолжается, система audio пишется как есть
- В UI показывается оранжевая плашка: "⚠️ Mic unavailable — only other participants will be captured"

Обратный случай (system audio fail) — НЕ graceful: это критичная часть meeting recording, запись прерывается с ошибкой в `lastError`.

## 3. Hallucination filter {#hallucination-filter}

### 3.1. Обоснование {#hallucination-filter.rationale}

Whisper (обучен на YouTube) генерирует фразы-артефакты когда встречает тишину/шум:
- "Продолжение следует..." / "To be continued..."
- "Спасибо за просмотр" / "Thanks for watching"
- "Субтитры от Amara.org" / "Subtitles by..."
- "Subscribe" / "Подписывайтесь"

На 5-минутных чанках где пользователь молчит или звук тихий — Whisper выдаёт такой мусор вместо пустой строки.

### 3.2. Переиспользование {#hallucination-filter.reuse}

`TranscriptionCoordinator` уже имеет отлаженный фильтр для обычной записи:
- `isAlwaysHallucination(_ text: String) -> Bool` — токены которые НИКОГДА не правда (независимо от RMS)
- `isHallucination(_ text: String) -> Bool` — паттерны которые правдоподобны на тихом аудио
- `calculateRMS(_ samples: [Float]) -> Float` — энергия

Все три — `static` методы, `internal` visibility. Meeting pipeline переиспользует их напрямую без дублирования.

### 3.3. Пороги {#hallucination-filter.thresholds}

- RMS < 0.0005 → чанк пропускается целиком (чистая тишина)
- `isAlwaysHallucination(text) == true` → всегда выкидываем
- RMS < 0.003 И `isHallucination(text) == true` → выкидываем

Пороги эмпирические, совпадают с обычной записью.

## 4. UI контракт {#ui-contract}

### 4.1. Popover strip {#ui-contract.strip}

Появляется в попапе ТОЛЬКО если `settings.meetingRecordingEnabled == true`.

Состояния:
- `isRecording` — красный индикатор, таймер (TimelineView), STOP кнопка, waveform визуализация под strip (§4.3)
- `isStarting` — спиннер + "STARTING..."
- idle + обнаружен meeting app — "{APP} DETECTED" + RECORD
- idle + нет meeting app — "NO MEETING" + RECORD

### 4.2. MeetingTimer {#ui-contract.timer}

Использует `TimelineView(.periodic(from: .now, by: 1))` + `@State startDate`.

**НЕ использовать** `Timer.publish(...).autoconnect()` с `@State elapsed` — parent view (meetingStrip) ре-рендерится много раз в секунду от audio level updates, subscription на timer пересоздаётся и никогда не доживает до первого тика.

### 4.3. Waveform во время записи {#ui-contract.waveform}

Когда `meetingRecorder.isRecording == true` — под strip показывается `MeetingWaveform` (отдельный view от `AudioLevelWave` который используется обычной записью).

**Источник данных:** `meetingRecorder.audioBars` — уже микшированный уровень (max из mic + system, см. §2.3).

**Почему отдельный view, а не переиспользуем `AudioLevelWave`:**
- `AudioLevelWave` приватный в `MenuBarView.swift` (нужно унести в Components или создать брат-view)
- Meeting waveform имеет другой контекст — полоска сверху стрипа, красная цветовая схема (recording)
- Разные размеры: полноширинный wave занимает стандартно 50pt, для meeting strip хотим 32pt (компактнее)

**Поведение:** когда `isStarting` или idle — waveform скрыт. Появляется только при active recording. При `micOnlyMode` — waveform всё равно показывается (system audio идёт один).

**Идентификация: если mic молчит но колонки играют — waveform должен отражать активность.** Это значит брать max(mic, system) в MeetingRecorder — уже реализовано в `CombineLatest` binding.

### 4.4. Копирование/экспорт транскрипта {#ui-contract.copy-export}

В `InsightsView → Meetings` каждая карточка meeting'а должна давать пользователю:
- **Copy** — скопировать полный текст транскрипта в clipboard
- **Export** — сохранить как `.txt` файл через NSSavePanel
- **Delete** — удалить из SwiftData (с confirmation)

**Поведение:**
- Кнопки появляются по hover на карточке (компактнее, не засоряет UI в idle)
- Copy даёт визуальный feedback (Text "COPIED" на 1.2s, как в попапе `lastOutput`)
- Export по умолчанию предлагает имя `Meeting-YYYY-MM-DD-HHMM.txt`
- Delete спрашивает подтверждение через `.confirmationDialog`

**Формат экспорта:**
```
Meeting — [date] [time]
Duration: [X] min
Model: [modelName]

[transcript text]
```

**Файл:** `Views/Windows/InsightsView.swift` § `meetingCard`.

**Обоснование:** транскрипт созвона — это ценный артефакт. Без возможности вытащить его наружу (в Notion, Obsidian, email) фича бесполезна. Copy покрывает 90% кейсов, Export — для тех кто хочет файл.

## 5. Сегментация {#segmentation}

Whisper large-v3 принимает до 30 сек за раз (внутренне). Для длинных записей разбиваем на куски по 5 минут (16000 \* 300 = 4.8M samples).

```swift
let chunkSize = Int(16000 * 300)
chunks = stride(from: 0, to: samples.count, by: chunkSize).map {
    Array(samples[$0..<min($0 + chunkSize, samples.count)])
}
```

Транскрипты чанков склеиваются через `\n\n`.

## 6. Engine selection {#engine-selection}

НЕ создавать отдельный TranscriptionCoordinator для meeting. Использовать `coordinator.activeEngine` — он уже знает текущий выбор on-device / cloud и загруженную модель. Это убирает класс ошибок "Meeting coord's engine устарел".
