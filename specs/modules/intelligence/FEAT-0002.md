# FEAT-0002: Screen Context {#root}

## User story {#story}

**Как** пользователь,
**Я хочу** чтобы MetaWhisp понимал контекст моей работы на Mac (какое приложение, какое окно, какой текст на экране) → чтобы давать релевантные советы (FEAT-0003).
**Без** отправки скриншотов в облако.

## Acceptance criteria (Critical) {#acceptance}

- Включается тоглом в Settings → Intelligence → Screen Context
- При включении: триггерится диалог Screen Recording permission (если ещё нет)
- Захват срабатывает при СМЕНЕ окна (не каждые N секунд), чтобы не дёргать систему зря
- OCR работает полностью on-device через Apple Vision
- **Скриншоты НИКОГДА не сохраняются на диск** — обрабатываются in-memory и GC'шатся
- Blacklist приложений срабатывает до capture (банки, пароли, мессенджеры не трогаются)
- Persistence: текст + app name + window title + timestamp в SwiftData
- Отключение тоглом → сервис останавливается БЕЗ перезапуска app

## 1. Pipeline {#pipeline}

```
NSWorkspace.frontmostApplication changes
    ↓
ScreenContextService.captureIfChanged()
    ├─ bundleID in blacklist? → skip
    ├─ whitelist mode & not in list? → skip
    ├─ same app+window as last? → skip
    └─ proceed:
        ├─ SCShareableContent.current → SCScreenshotManager.captureImage
        ├─ Vision VNRecognizeTextRequest(.accurate)
        └─ persist: ScreenContext(appName, windowTitle, ocrText)
```

## 2. Инварианты приватности {#privacy}

- CGImage существует только между capture и OCR — после ocrText извлечён, изображение больше не referenced
- SwiftData хранит ТОЛЬКО text + metadata
- Default blacklist: `com.apple.Passwords`, `com.apple.keychainaccess`, 1Password, Bitwarden
- Пользователь может дополнить blacklist через Settings (spec §6)
- Whitelist режим: если задан — ТОЛЬКО перечисленные apps захватываются

### 2.1. Намеренный дивергенс от референса {#privacy.divergence}

**В референсной реализации blacklist/whitelist отсутствует** — она скриншотит всё подряд без user-facing фильтра. MetaWhisp осознанно расширяет приватность:
- Default blacklist защищает password managers / Keychain даже если пользователь не трогал Settings
- Конфигурируемый whitelist позволяет параноидальному пользователю capture ТОЛЬКО выбранные apps (например: только терминал + IDE для dev-советов, никаких мессенджеров)

Это **фича, не баг** — не удалять при next-session рефакторинге "для паритета с референсом".

## 3. Apple Vision OCR {#ocr}

`VNRecognizeTextRequest` с:
- `recognitionLevel = .accurate` (не fast — качество важнее скорости, и так бежит раз в N секунд)
- `usesLanguageCorrection = true`
- `recognitionLanguages = ["en-US", "ru-RU", "de-DE", "fr-FR", "es-ES"]`
- `automaticallyDetectsLanguage = true`

Возвращает массив `VNRecognizedTextObservation`, берём `topCandidates(1).first?.string`, склеиваем через `\n`.

## 4. Permission handling {#permission}

Первый вызов `SCShareableContent.current` триггерит ScreenCaptureKit permission диалог. Если пользователь денаёт — capture returns nil, но monitoring не падает (graceful degradation — остаются metadata без OCR).

Для первого start: `PermissionsService.requestScreenRecording()` вызывается ПРЕЖДЕ чем стартует цикл мониторинга. Если разрешение не выдано — `isActive = false`, System Settings открываются.

## 5. SwiftData модель {#model}

```swift
@Model final class ScreenContext {
    var id: UUID
    var timestamp: Date
    var appName: String
    var windowTitle: String
    var ocrText: String
    var summary: String?  // резерв под LLM-summary (не заполняется сейчас)
}
```

Persistence через общий `HistoryService.modelContainer`.

## 6. UI для blacklist/whitelist {#app-picker}

### 6.1. Проблема {#app-picker.problem}

Сейчас в Settings → Intelligence → Screen Context список приложений — **comma-separated TextField** (`screenContextAppList`). Пользователю нужно знать точные названия/bundle ID и вводить их вручную. Неудобно и error-prone.

### 6.2. Контракт {#app-picker.contract}

- Показывать **установленные приложения с Mac** (через `NSWorkspace.shared.runningApplications` + сканирование `/Applications` и `~/Applications`)
- Каждое приложение: icon + name + bundle ID
- Добавление: кнопка `+` → модальный picker со списком приложений
- Удаление: кнопка `✕` рядом с каждым выбранным приложением
- Список хранится в `AppSettings.screenContextAppList` как comma-separated (сохраняем существующий формат для обратной совместимости)
- Ключом для хранения/сравнения использовать **bundle ID** (`com.apple.Safari`), не localizedName (он меняется с локалью)

### 6.3. Режим (blacklist/whitelist) {#app-picker.mode}

`screenContextMode` (existing) определяет семантику:
- `"blacklist"` — перечисленные apps **исключаются** (default behavior)
- `"whitelist"` — захватываются **только** перечисленные apps

UI один и тот же для обоих режимов, но label меняется: "EXCLUDED APPS" / "INCLUDED APPS".

### 6.4. Источник установленных приложений {#app-picker.source}

Для списка "что можно добавить" используем:

1. **Running applications** — `NSWorkspace.shared.runningApplications` (cheap, instant)
2. **Installed apps** — сканирование `/Applications/*.app` и `~/Applications/*.app` (kept-alive in @State cache)

Оба источника объединяются по bundle ID (dedup). Фильтр: только apps с `bundleIdentifier != nil` и не system daemons.

### 6.5. Файлы {#app-picker.files}

- `Views/Components/AppPickerView.swift` — новый компонент (sheet)
- `Views/Components/AppPickerRow.swift` — строка списка (icon + name + bundleID)
- `Views/Windows/MainSettingsView.swift` § `intelligenceSection` — заменить TextField на новый UI
- `Helpers/InstalledApps.swift` — утилита для сканирования (может быть внутри AppPickerView если не нужно переиспользовать)

## 7. UI индикатор активности {#ui-indicator}

### 7.1. Проблема {#ui-indicator.problem}
Пользователь не видит работает ли Screen Context в данный момент. Тогл в Settings — далеко, а попап menu bar показывает только Meeting Recording. Screen Context работает в фоне невидимо → пользователь думает "сломалось".

### 7.2. Контракт {#ui-indicator.contract}
В попапе menu bar добавить **compact strip** (аналогично `meetingStrip`):
- Показывается ТОЛЬКО если `settings.screenContextEnabled == true`
- Иконка 👁 или `eye` SF Symbol + "SCREEN CONTEXT"
- Последнее захваченное: `{appName} — {windowTitle}` (truncated)
- Если capture ещё не было — "WATCHING..."
- Не занимает много места (одна строка)

### 7.3. Observable источник {#ui-indicator.source}
`ScreenContextService.lastContext` уже `@Published`. Реактивно обновляется при каждом новом capture. Popover подписывается через `@ObservedObject`.

**Файл:** `Views/MenuBar/MenuBarView.swift` § новый `screenContextStrip`

## 8. Memory footprint {#memory}
