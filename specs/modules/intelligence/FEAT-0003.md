# FEAT-0003: AI Advice {#root}

## User story {#story}

**Как** пользователь,
**Я хочу** чтобы MetaWhisp поверх моей активности (screen context + транскрипции) генерировал релевантные подсказки — продуктивность, здоровье, коммуникация, обучение.
**Используя** уже настроенный LLM provider (не отдельный).

## Acceptance criteria (Critical) {#acceptance}

- Включается тоглом Settings → Intelligence → AI Advice
- Триггеры: периодический (каждые N минут, default 15) + сразу после успешной транскрипции
- Требует API ключ в LLM Provider (или Pro подписку)
- Промпт включает: последние 5 screen contexts + опционально свежая транскрипция
- Ответ парсится как JSON: `{content, category, reasoning, confidence}`
- Categories: productivity / health / communication / learning
- В Insights → AI Advice показываются карточки с dismiss + mark-read
- **НЕ отправляется** в отдельный сервис — используется тот же `OpenAIService` что и обычная обработка

## 1. Архитектура {#architecture}

```
Settings toggle ON
    ↓
AdviceService.configure(screenContext, modelContainer)
AdviceService.startPeriodicAdvice(interval: 15min)
    ↓
[every 15 min OR after transcription]
    ↓
generateAdvice(extraContext?)
    ├─ читает screenContext.recentContexts (до 20 в памяти)
    ├─ buildContextSummary → промпт строка
    ├─ OpenAIService.complete(system: advicePrompt, user: summary, apiKey, provider)
    ├─ parseAdviceResponse(JSON)
    └─ SwiftData insert AdviceItem
```

## 2. SwiftData модель {#model}

```swift
@Model final class AdviceItem {
    var id: UUID
    var content: String
    var category: String  // productivity | health | communication | learning
    var reasoning: String?
    var sourceApp: String?
    var confidence: Double  // 0.0-1.0
    var isRead: Bool
    var isDismissed: Bool
    var createdAt: Date
}
```

## 3. Промпт {#prompt}

System:
```
You are a productivity assistant embedded in a macOS app called MetaWhisp.
Based on the user's recent screen activity, provide ONE concise, actionable
piece of advice. Focus on productivity, health (breaks, posture),
communication, or learning. Be specific and contextual.

Respond in JSON format:
{"content": "...", "category": "...", "reasoning": "...", "confidence": 0.0-1.0}
```

User: контекст последних 5 screen contexts + optional transcription excerpt.

## 3.5. Prompt engineering {#prompt}

### 3.5.1. Anti-repetition {#prompt.anti-repetition}

Без памяти предыдущих советов LLM склонен повторять одно и то же (наблюдалось: "Take a 5-10 min break to stretch..." три раза подряд при одинаковом контексте).

**Решение:** передавать **последние N советов** (last 5, <3 дней) в промпт как "уже было, не повторяй":

```
Previous advice (do NOT repeat these themes, pick a different angle):
- [category] content (time ago)
- [category] content (time ago)
```

**Вариативность через temperature:** 0.5 → 0.7-0.8 (Groq llama-3.3). Повышает креативность без ухудшения структуры.

### 3.5.2. Rotate categories {#prompt.categories}

Даже с уникальным контентом, если 80% советов одной категории (health), это ощущается повтором.

**Решение:** в промпт передать "recent categories: health, health, health" и потребовать от модели **explicitly выбрать другую** если 2+ последних советов одной категории.

### 3.5.3. Rich context {#prompt.rich-context}

Сейчас промпт содержит только последние 5 screen contexts. Добавить:
- **Последние 3-5 транскрипций** (если есть) — даёт тон разговоров пользователя
- **Время суток** — утром уместнее productivity, вечером — health/wind-down
- **Длительность сессии** — сколько минут в текущем app

Это **не** поведенческая аналитика (нет персональных stats), а просто более богатый mise-en-scène для LLM.

### 4.1. Pro users (через server proxy) {#llm.pro}

Для Pro пользователей используется `POST https://api.metawhisp.com/api/pro/advice` — endpoint на Cloudflare Worker (api/src/index.js § `handleProAdvice`):
- Auth: `Bearer {licenseKey}` (валидируется `validateProLicense`)
- Body: `{ system: string, user: string }`
- Response: `{ text: string }` — JSON с advice (парсится на клиенте)
- Server-side: вызов Groq `llama-3.3-70b-versatile` с `GROQ_API_KEY` секретом, temperature=0.5, max_tokens=1024
- Length cap: 8000 символов на каждое из полей (anti-abuse)

### 4.2. Non-Pro (свой API ключ) {#llm.direct}

Прямой вызов `OpenAIService.complete()` с ключом из Keychain (`settings.activeAPIKey`). Провайдер из `AppSettings.llmProvider`:
- `openai` — GPT-4o-mini
- `cerebras` — Qwen 3 235B

### 4.3. Выбор пути {#llm.routing}

В `AdviceService.generateAdvice()`:
```swift
if LicenseService.shared.isPro, let licenseKey = LicenseService.shared.licenseKey {
    response = try await callProProxy(system:, user:, licenseKey:)
} else {
    guard !settings.activeAPIKey.isEmpty else { return nil }
    response = try await llm.complete(system:, user:, apiKey:, provider:)
}
```

`hasLLMAccess` = `!activeAPIKey.isEmpty OR isPro` — общий guard для `triggerOnTranscription` и `generateAdvice`.

## 5. Триггеры {#triggers}

### 5.1. Периодический {#triggers.periodic}
`Task { while !cancelled { sleep(interval); generate() } }`. Запускается при startMonitoring, останавливается при stop.

### 5.2. По транскрипции {#triggers.transcription}

`triggerOnTranscription(text, source)` — вызывается из `TranscriptionCoordinator` после успешной транскрипции (не галлюцинации). Добавляет excerpt транскрипции как extraContext.

**Контракт:**
- Вызывается ПОСЛЕ фильтрации галлюцинаций (только на "настоящих" транскриптах)
- Вызывается ПОСЛЕ `textProcessor.process()` чтобы использовать очищенный текст, а не сырой Whisper output
- Вызывается ПОСЛЕ `historyService.save()` (если нужен — history item не критичен для advice генерации)
- `source` = "microphone" для обычной записи, "meeting" для meeting recording
- Fire-and-forget: coordinator не ждёт завершения advice генерации (она асинхронна, может занимать секунды)
- Не вызывается для слишком коротких текстов (< 20 символов) — нет смысла генерировать совет из одного слова

**Интеграция:**
- `TranscriptionCoordinator` держит опциональный `adviceService: AdviceService?` (может быть nil если advice выключен)
- `AppDelegate` прокидывает ссылку при старте coordinator
- Аналогично для meeting pipeline — `stopMeetingRecording` в AppDelegate вызывает `adviceService.triggerOnTranscription` после успешной транскрипции meeting'а

**Почему не через notification / publisher:** coordinator и advice живут в одном процессе, прямой call проще; notifications добавили бы лишнюю indirection без выгоды.

## 5.3. macOS нотификации {#notifications}

### 5.3.1. Why {#notifications.why}

Без нотификаций пользователь должен вручную открывать Insights → AI Advice чтобы увидеть новый совет. Это обнуляет ценность периодических advice — пока он не откроет таб, советы копятся впустую.

Референс решает это системными нотификациями + floating bar (отдельная surface). MetaWhisp пока без floating bar — ограничиваемся системными нотификациями macOS через `UNUserNotificationCenter`.

### 5.3.2. Контракт {#notifications.contract}

- **Триггер:** каждый новый `AdviceItem` добавленный в SwiftData → посылается нотификация
- **Title:** category (capitalized): "Productivity", "Health", "Communication", "Learning"
- **Body:** `item.content` truncated to ~200 символов
- **Sound:** default macOS notification sound
- **Click action:** app выходит на передний план + открывает Insights tab в main window + помечает item как `isRead = true`
- **Permission:** запрашивается **при первом включении AI Advice toggle** (не при запуске app)
- **Rate limit:** не более 1 нотификации в минуту (против spam — если LLM внезапно выдаст несколько подряд)
- **Dismiss tracking:** сейчас не нужно — этого хватит для MVP

### 5.3.3. Почему не рассылать каждый периодический trigger {#notifications.periodicity}

Периодический trigger (каждые 15 мин) **уже rate limit** — не более 1 advice за interval. Rate limit на уровне NotificationService — защита от двойного trigger (периодический + транскрипция близко по времени).

### 5.3.4. Архитектура {#notifications.arch}

```
AdviceService.generateAdvice()
    ↓
    ... парсим JSON ...
    ↓
    modelContext.insert(advice)
    ↓
    NotificationService.shared.postAdvice(advice)  ← new
    ↓
    UNUserNotificationCenter.current().add(request)
```

**Файл:** `Services/System/NotificationService.swift` — singleton, обёртка над `UNUserNotificationCenter` + UNUserNotificationCenterDelegate для обработки click.

**Hook на click:** на `didReceive` делегата → NotificationCenter broadcast `.openInsightsForAdvice` + post `.switchMainTab` с `.insights` + отметить item как read.

### 5.3.5. Permission UX {#notifications.permission}

При включении toggle "AI ADVICE" в Settings — если notifications не granted:
1. Триггерим `UNUserNotificationCenter.current().requestAuthorization(options: [.alert, .sound, .badge])`
2. Если denied — toggle остаётся ON (advice всё равно работает, просто без уведомлений), но показываем inline hint "Enable notifications in System Settings for advice popups"

Не блокируем сам advice если permission не выдано — пользователь всё равно увидит в Insights tab.
