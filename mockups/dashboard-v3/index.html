Dashboard v3 — 5 mockups

Каждый вариант фиксит: layout-jump при смене дня, tomorrow с реальными планами (events / due tasks / suggested), и no-empty-day frustration. Click чтобы открыть. Скажи какой нравится — портирую в приложение.

VARIANT 1
Feed (Omi-style)
Single chronological scroll: today → yesterday → past. Каждый день — full card. Никакого picker'а.
+ No layout jump · + familiar timeline UX · + tomorrow в начале как pinned card
VARIANT 2
macOS Mail (sidebar list + main detail)
Слева — список дней (date + headline preview). Справа — main detail fixed-height со scroll внутри.
+ Detail area FIXED — никаких прыжков · + compact navigation · + familiar pattern
VARIANT 3
TODAY + TOMORROW split (2 columns)
Главное: today слева большой + tomorrow plans справа. Past — отдельный compact link below.
+ Future-aware UI · + tomorrow first-class · + past not in face
VARIANT 4
3-day window (Yesterday · Today · Tomorrow)
Три card бок о бок: вчера / сегодня (главный, увеличенный) / завтра. Past — collapse below.
+ Always 3 — no jump · + прошлое + будущее одновременно
VARIANT 5
Today fullscreen + collapsed past
Today доминирует весь экран. Past 7 days — collapsed accordion внизу. Tomorrow — sticky banner сверху.
+ Maximum focus on today · + tomorrow always visible · + past — opt-in

Reference: Omi пользует single chronological feed для memory items — нет calendar/picker концепции. v1 ближе всего к этому. Но я также сделал альтернативы (sidebar / split / 3-day / focus) на случай если для desktop хочется другой подход.
